# S3

## Overview

Use `aws.S3(bucketName, options?)` for S3 object operations. The README documents standard file upload and download, delete and move, text helpers, streaming lines, and presigned upload/download URLs.

## Setup Pattern

```ts
import * as fs from "fs";
import { AntokelAws } from "antokel-cloud";

const aws = new AntokelAws();
const s3 = aws.S3("my-bucket");
```

Prefix usage from the README:

```ts
const s3 = aws.S3("my-bucket", { prefix: "folder/subfolder" });
```

Trailing slashes in the prefix are accepted and normalized. Use this when the user wants all operations scoped under a logical folder.

## Common Tasks

### Upload, Download, Move, Delete

```ts
await s3.upload(fs.readFileSync("./local/file.pdf"), "path/on/s3.pdf");
await s3.download("path/on/s3.pdf", "./local/file.pdf");
await s3.move("old/path.pdf", "new/path.pdf");
await s3.remove("path/on/s3.pdf");
```

If the user asks to upload raw bytes or a file buffer, use `s3.upload(source, key)`.

### Download as Base64 or Bytes

```ts
const asDataUrl = await s3.download("path/on/s3.pdf", s3.as.base64);
const asBuffer = await s3.download("path/on/s3.pdf", s3.as.bytes);
```

Use `s3.as.base64` when the user needs a data URL string. Use `s3.as.bytes` when the user needs a `Buffer`.

### Read and Write Text

```ts
const content = await s3.asText.read("notes.txt");
await s3.asText.write("hello world", "notes.txt");
```

Use `asText` for plain text workflows instead of manual buffer conversion.

### Stream Text Lines

```ts
for await (const line of s3.asText.streamLines("data.csv")) {
  console.log(line);
}
```

The README explicitly notes that this streams lines with a single S3 GET billed. Use this pattern for large text files when the user wants iterative processing.

## Presigned URLs

Use `s3.presigned.upload(...)` to create a browser upload target and `s3.presigned.download(...)` to create a download URL.

```ts
const upload = await s3.presigned.upload("path/to/file.webp", {
  contentType: selectedImage.type,
  expiresInSeconds: 1800, // 30 minutes
});

const uploadResponse = await fetch(upload.url, {
  method: upload.method,
  headers: upload.headers,
  body: selectedImage,
});

if (!uploadResponse.ok) {
  throw new Error(`Upload failed with status ${uploadResponse.status}`);
}

const { bucket, pathToFile } = upload;
const getUrl = await s3.presigned.download(pathToFile, {
  expiresInSeconds: 1800, // 30 minutes
});
```

If the user needs temporary access, set `expiresInSeconds`. The SDK defaults to `900` seconds (15 minutes) and S3 presigned URLs should not exceed `604800` seconds (7 days).

## Important Caveats

- Use `PUT` for browser uploads.
- Do not propose `fetch(..., { method: "POST", body: file })` as a presigned upload flow.
- Do not claim S3 returns JSON from the upload request. The README explicitly says it does not.

## Request Mapping

- “Upload a local file to S3” -> `s3.upload(...)`
- “Download to disk” -> `s3.download(key, localPath)`
- “Get the file as base64” -> `s3.download(key, s3.as.base64)`
- “Get the file as bytes” -> `s3.download(key, s3.as.bytes)`
- “Read or write text files” -> `s3.asText.read(...)` or `s3.asText.write(...)`
- “Process a large text object line by line” -> `s3.asText.streamLines(...)`
- “Generate browser upload/download links” -> `s3.presigned.upload(...)` and `s3.presigned.download(...)`
